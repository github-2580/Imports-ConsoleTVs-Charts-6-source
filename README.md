This public function is present in the chart calss. So it's accesible from the chart <code>__construct()</code> or in the controller iself. This option is the only one you'll need to configure the chart at your needs. It's basically an array that is later converted into a JS object, and appended correctly in the chart library. It stores the global chart configuration. </p><div class="bg-orange-lighter border-l-8 border-orange py-2 px-4 mb-2"> Do not get confused between the chart options and the dataset options! </div><p class="py-2"> To show how this works, we'll change the example chart we created before, by configuration the chart a bit. </p><div class="bg-blue-lighter border-l-8 border-blue py-2 px-4 mb-2"> While you learn how to customize the chart without any presets, please understand you'll need to know what options does the library you're working with give you. Theese includes: </p><ul><li><code>dataset(string $name, string $type, $data)</code> - Adds a new dataset to the chart.</li><li><code>labels($labels)</code> - Set the chart labels.</li><li><code>options($options, bool $overwrite = false)</code> - Set the chart options.</li><li><code>container(string $container = null)</code> - Set the chart container (if you need your own view). Display the view if no parameter is set.</li><li><code>type(string $type)</code> - Force a chart type, if not set, the first dataset type found will be used.</li><li><code>height(int $height)</code> - Set the chart height. Null / 0 by default = auto</li><li><code>width(int $width)</code> - Set the chart width. Null / 0 by default = auto</li><li><code>loader(bool $loader)</code> - Determines if the chart loader should be displayed. In fact, if we want, for example, to change the color of the line chart to a diferent one (we're still using the chartjs example) we would see in their documentation that the line chart have a 'borderColor' option.
